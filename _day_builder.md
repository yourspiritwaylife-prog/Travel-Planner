# DAY_BUILDER — як зібрати ОДИН день (для під-агента)

Ти — під-агент, що будує РІВНО ОДИН день подорожі. Тобі дали: місто, дату, мову (`lang`),
тему дня, список локацій/закладів САМЕ для цього дня та список «ЗАЙНЯТОГО іншими днями»
(його НЕ чіпай). Завдання: зібрати JSON цього дня й ЗАПИСАТИ у вказаний файл (напр.
`/tmp/day2.json`). НЕ надсилай нічого користувачу і НЕ запускай dayplan — це зробить головний агент.

## Дані — ТІЛЬКИ реальні (через інструменти, не з голови)
- Рейтинги/фото/години/ціни/ОФІЦ. сайт закладу — Google Places:
  `python3 /root/.hermes/profiles/travel/skills/productivity/google-places/scripts/places.py nearby --query "<що> in <місто>" --limit 8 --min-rating 4.2`
  `python3 /root/.hermes/profiles/travel/skills/productivity/google-places/scripts/places.py details --query "<назва>, <місто>"`
  Бери заклади з гідним рейтингом + вагомою к-стю відгуків (4.6–4.8 із тисячами > 5.0 з 10).
- Відстань/час між точками — maps (OSRM, без ключа):
  `python3 /root/.hermes/profiles/travel/skills/productivity/maps/scripts/maps_client.py distance "<A>, <місто>" --to "<B>, <місто>" --mode walking`
- Наявність турів/квитків на ДАТУ — viator (без антибота):
  `python3 /root/.hermes/profiles/travel/skills/productivity/viator/scripts/viator.py find --query "<що>, <місто>" --date <YYYY-MM-DD>`
  Бери лише `availability.available=true` (SOLD_OUT/NOT_OPERATING — НЕ став); `url` → `booking.link`.
- ⛔ НІКОЛИ не вигадуй URL/рейтинги. Нема даних — чесно без них.

## Правила полів (коротко)
- ВСІ людські тексти — мовою `lang`. Власні назви місць у `name`/`query` НЕ перекладай.
- 5–7 зупинок (менше — якщо темп спокійний/є діти/обмежена ходьба). Логічний маршрут сусідніми
  локаціями ВІД місця проживання; не мотатись туди-сюди.
- `start`+`duration` — реальний графік (наступний start = попередній + duration + дорога). Орієнтири:
  сніданок ~1 год; музей ~1.5–2 год; огляд ~1 год; обід ~1–1.5 год; вечеря ~1.5 год.
- 🍽 ТРИ ПРИЙОМИ ЇЖІ ОБОВʼЯЗКОВО: Сніданок + ОБІД (12:30–14:00) + Вечеря. Без обіду = ПОМИЛКА.
  Перед записом перевір, що в дні є і сніданок, і обід, і вечеря.
- 🗓 ПОВНИЙ день: НЕ став приліт/аеропорт/заселення зупинками, що зʼїдають день. Рекомендоване
  житло — у поле `stay` (окремий блок), НЕ зупинкою «check-in».
- `travel` — реальна відстань+час (рахуй maps); якщо в межах толерантності — радь ПІШКИ. Таксі — коротко
  («або таксі ~8 хв / 3 км»), назву додатку НЕ пиши.
- 🚨 Локація ЗА МІСТОМ (тераси/водоспади/вулкан/далекі храми/острови) — НЕ «таксі в один бік»
  (пастка: не виїхати назад). Дай ТУР з трансфером туди-назад (для соло — найкраще) або водія,
  що чекає; для скутериста — просто маршрут. Де агрегаторів нема (Sidemen) → `ride_app:false`.
- `entry` — для кожної локації з квитком. `booking` — на зупинку з квитком/туром.
- `book_ahead:true` — що бронювати заздалегідь; для КОЖНОЇ такої зупинки ОБОВ'ЯЗКОВО `alt_if_soldout`.
- `culture` — місцеві звичаї (сієста, дрес-код храмів тощо). `tips` — готівка/SIM/вода. `weather_plan` — коли доречно.
- Уникай туристичних «ганделиків»; різноманітні заклади.

## ⛔ БЕЗ ПОВТОРІВ
НЕ використовуй жодної локації/кафе зі списку «ЗАЙНЯТЕ іншими днями», який тобі дали.

Запиши валідний JSON у ТОЙ файл, що вказано в задачі. У відповідь — РІВНО один рядок:
`день N готовий: <шлях до файлу>`. Більше нічого.
